❌ Fails: All direct route URLs return 404

ROOT CAUSE:
nginx is not configured to serve index.html for routes that don't match files.
React Router handles routing on the client side, so all routes need to serve 
the same index.html file.

REQUIRED FIX:
Please add this configuration to my nginx server block for shraddhainstitute.in:

    location / {
        try_files $uri $uri/ /index.html;
    }

EXPLANATION OF FIX:
- try_files $uri: First, try to serve the requested file
- $uri/: If not a file, try as a directory
- /index.html: If neither, serve index.html (React app takes over)

This is a standard configuration for Single-Page Applications and is required
for React, Vue, Angular, and similar frameworks.

VERIFICATION AFTER FIX:
Once configured, http://www.shraddhainstitute.in/CompetitionLandingPage 
should return HTTP 200 (not 404) and serve the index.html file.
